# delivery-system テスト設計書

## 1. テスト方針

### 目的
- 各画面の正常系・異常系の動作を確認する
- 意図的に仕込んだバグをテストで検知する
- 回帰テストとしてCIに組み込み、継続的にバグを検知できる状態にする

### テスト種別
| 種別 | 内容 |
|------|------|
| 機能テスト | 画面・機能単位で正常系・異常系を確認 |
| シナリオテスト | ユーザーの一連の操作を通しで確認 |

### テスト自動化方針
- ✅ Playwright を使用したE2Eテスト
- ✅ Page Object Model（POM）を導入し、保守性を高める
- 📋 GitHub Actions でCIに組み込み、pushのたびに自動実行する

### 本テスト設計のスコープ

本ポートフォリオは、QAスキル（テスト設計・自動化・CI組み込み）の本質を示すことを目的とし、以下を実装スコープから除外している。

- 入力値の文字数・文字種バリデーション
- セキュリティ関連のテスト
- カート内の数量変更・削除機能のテスト

実務であれば必須となる観点については、本書末尾の「5. 本来必要な追加観点（実装スコープ外）」に明記している。

## 3. シナリオテスト

### シナリオ①：新規ユーザーの購入フロー（正常系）

```
①会員登録画面で新規登録する
↓
②ログイン画面でログインする
↓
③商品一覧画面で商品をカートに追加する
↓
④注文確認画面で内容を確認して注文する
↓
⑤注文履歴画面で注文内容を確認する
```

**確認ポイント**
- 各画面の遷移が正しいこと
- 注文履歴に正しい内容が表示されること
- 到着予定日が注文日から3日後であること

---

### シナリオ②：バグ①の再現（上書きバグ）

```
①ログインする
↓
②商品Aを注文する → 注文履歴に表示されることを確認
↓
③商品Bを注文する
↓
④注文履歴画面で商品Aの履歴が消えていることを確認（バグ検知）
```

**期待される正しい動作**
- 商品Aと商品Bの両方の注文履歴が表示される

---

### シナリオ③：バグ②の再現（境界値バグ）

```
①ログインする
↓
②合計金額がちょうど3000円になるように商品を選択する
③注文確認画面で送料300円が表示されることを確認（バグ検知）
```

**期待される正しい動作**
- 3000円以上で送料無料になる

## 4. 自動化優先度

| 優先度 | テストケース | 理由 |
|--------|-------------|------|
| 高 | シナリオ①②③ | バグ検知・回帰テストの核心部分 |
| 高 | R-01, L-01, P-08, OC-06, OH-06 | 主要な正常系フロー |
| 中 | R-02〜R-08, L-02〜L-05 | 異常系・バリデーション確認 |
| 低 | P-01〜P-03 | データ確認系（変更頻度低） |

## 5. 本来必要な追加観点（実装スコープ外）

本ポートフォリオはQA自動化スキルを示すことに集中しており、以下の観点は実装スコープから外しているが、実務であれば必須となる観点として明記しておく。

### 5-1. バリデーション・入力制御

| 画面 | 観点 |
|------|------|
| 会員登録 | 名前の文字数上限・最小値の境界値テスト |
| 会員登録 | メールアドレス形式の妥当性チェック（@なし、不正文字含む等） |
| 会員登録 | 電話番号の文字種チェック（数字以外を弾く、ハイフンの有無） |
| 会員登録 | パスワードの強度チェック（最小文字数・記号・大文字小文字混在） |
| 商品一覧 | 数量入力欄に0・負数・小数・文字列を入力した場合の挙動 |
| 商品一覧 | 数量の上限値（在庫数を超えた場合の制御） |

### 5-2. セキュリティ

| 観点 | 内容 |
|------|------|
| SQLインジェクション | 入力値にSQL文を含めた場合に攻撃が成立しないこと |
| XSS（クロスサイトスクリプティング） | 入力値にスクリプトを含めた場合に実行されないこと |
| パスワードのハッシュ化 | DBに平文でパスワードが保存されていないこと |
| セッション管理 | localStorageでのユーザーID保持は本来JWTやセッションIDに変更すべき |
| CSRF対策 | 注文・キャンセルAPIへの不正リクエスト対策 |

### 5-3. 異常系・エッジケース

| 観点 | 内容 |
|------|------|
| 通信エラー | API通信失敗時のエラーハンドリング・リトライ |
| タイムアウト | 長時間応答がない場合の挙動 |
| 同時実行 | 複数タブで同時注文した場合の挙動 |
| ブラウザ戻る | 注文完了後にブラウザバックした場合の挙動 |
| localStorage削除 | カート保持中にlocalStorageが消えた場合の挙動 |

### 5-4. 機能面の追加観点

| 画面 | 観点 |
|------|------|
| 商品一覧 | 在庫切れ商品の表示・カート追加不可制御 |
| 商品一覧 | カート内の数量変更・個別削除 |
| 注文確認 | 前画面に戻るボタンでカート内容が保持されること |
| 注文履歴 | 注文ステータス（発送済み・配達中・完了など）の表示 |
| 注文履歴 | キャンセル可能期限の制御 |

### 5-5. 非機能要件

| 観点 | 内容 |
|------|------|
| パフォーマンス | 商品一覧の表示速度（100件・1000件時の挙動） |
| アクセシビリティ | スクリーンリーダー対応・キーボード操作 |
| レスポンシブ | スマホ・タブレット表示時のレイアウト崩れ |
| 多言語対応 | 英語・中国語等への切替 |
| ブラウザ互換性 | Chrome・Safari・Firefox・Edgeでの動作確認 |
