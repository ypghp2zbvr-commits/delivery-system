# delivery-system 機能仕様書

## 1. システム概要

果物を販売するECサイトのデリバリー管理システム。  
QAポートフォリオとして、意図的にバグを仕込み、テスト設計・自動化・CI組み込みまでを一貫して実施する。

| 項目 | 内容 |
|------|------|
| システム名 | delivery-system |
| 技術スタック | Next.js / TypeScript / SQLite / Playwright / GitHub Actions |
| 対象ユーザー | 会員登録済みのユーザー |

### 本ポートフォリオのスコープ

本システムはQAスキル（テスト設計・自動化・CI組み込み）を示すことを主目的としており、以下の機能は実装スコープから除外している。

- 入力値の文字数・文字種バリデーション
- パスワードのハッシュ化などのセキュリティ強化
- カート内の数量変更・削除機能
- 画面間の戻るボタンなどの一部ナビゲーション
- 商品在庫管理・在庫切れ表示
- 注文ステータス管理（発送済み・配達中など）

実務であればこれらも仕様として必要だが、本ポートフォリオではテスト自動化の本質を示すために簡略化している。なお、これらの観点は「テスト設計書」内に「本来必要な追加観点」として明記している。

## 3. 画面仕様

### 3-1. 会員登録画面（/register）

**入力項目**

| 項目 | 型 | 必須 | 備考 |
|------|----|------|------|
| 名前 | テキスト | ○ | |
| 住所 | テキスト | ○ | |
| 電話番号 | テキスト | ○ | |
| メールアドレス | メール | ○ | 重複不可 |
| パスワード | パスワード | ○ | |

※全項目の文字数・文字種制限は本ポートフォリオのスコープ外とする

**ボタン**
- 登録するボタン：入力内容をAPIに送信し、成功したらログイン画面へ遷移
- ログインリンク：ログイン画面へ遷移

**バリデーション**
- 全項目未入力の場合：「全項目を入力してください」エラー表示
- 既存メールアドレスの場合：「このメールアドレスは既に登録されています」エラー表示

---

### 3-2. ログイン画面（/login）

**入力項目**

| 項目 | 型 | 必須 |
|------|----|------|
| メールアドレス | メール | ○ |
| パスワード | パスワード | ○ |

**ボタン**
- ログインボタン：認証成功後に商品一覧画面へ遷移
- 会員登録リンク：会員登録画面へ遷移

**バリデーション**
- 未入力の場合：「メールアドレスとパスワードを入力してください」エラー表示
- 認証失敗の場合：「メールアドレスまたはパスワードが違います」エラー表示

---

### 3-3. 商品一覧画面（/products）

**表示内容**
- 商品名・価格・数量入力欄・カートに追加ボタン
- 1ページあたり10件表示
- ページネーションボタン

**操作**
- 数量入力：最小値1、初期値1
- カートに追加ボタン：同一商品は数量を合算してカートに追加
- 注文確認へボタン：カートが空の場合はアラート表示、カートの中身をlocalStorageに保存して注文確認画面へ遷移
- 注文履歴へボタン：注文履歴画面へ遷移

※ 数量の上限・0以下入力時の制御、カート内の数量変更・削除機能は本ポートフォリオのスコープ外とする

---

### 3-4. 注文確認画面（/order-confirm）

**表示内容**
- カートの商品一覧（商品名・数量・小計）
- 小計
- 送料
- 合計金額

**送料ロジック（※バグ仕込みあり）**

| 条件 | 送料 |
|------|------|
| 小計が3000円より大きい（> 3000） | 無料 |
| 小計が3000円以下（≤ 3000） | 300円 |

※本来の仕様は「3000円以上（>= 3000）で送料無料」が正しい。3000円ちょうどで送料がかかるバグを意図的に仕込んでいる。

**ボタン**
- 注文するボタン：注文APIを呼び出し、成功したらカートを削除して注文履歴画面へ遷移

※ 前画面への戻るボタンは本ポートフォリオのスコープ外とする

---

### 3-5. 注文履歴画面（/order-history）

**表示内容**
- 注文ごとの商品一覧（商品名・数量・小計）
- 注文日
- 到着予定日（注文日から3日後）
- 送料
- 合計金額

**操作**
- キャンセルボタン：確認ダイアログ表示後、注文を削除して一覧から除去
- 商品一覧に戻るボタン：商品一覧画面へ遷移

**ログイン未認証の場合**
- ページ表示時にlocalStorageのuserIdを確認し、未ログインであればログイン画面へ強制遷移

## 4. API仕様

### 4-1. 会員登録 POST /api/register

| 項目 | 内容 |
|------|------|
| リクエスト | name, address, phone, email, password |
| 成功レスポンス | 201 `{ message: '登録成功' }` |
| エラー（未入力） | 400 `{ error: '全項目を入力してください' }` |
| エラー（重複） | 400 `{ error: 'このメールアドレスは既に登録されています' }` |

### 4-2. ログイン POST /api/login

| 項目 | 内容 |
|------|------|
| リクエスト | email, password |
| 成功レスポンス | 200 `{ userId, userName }` |
| エラー（未入力） | 400 `{ error: 'メールアドレスとパスワードを入力してください' }` |
| エラー（認証失敗） | 401 `{ error: 'メールアドレスまたはパスワードが違います' }` |

### 4-3. 商品一覧取得 GET /api/products

| 項目 | 内容 |
|------|------|
| クエリパラメータ | page（ページ番号、デフォルト1） |
| 成功レスポンス | 200 `{ products: [...], total: number }` |

### 4-4. 注文作成 POST /api/orders

| 項目 | 内容 |
|------|------|
| リクエスト | userId, cart, totalPrice, shippingFee |
| 成功レスポンス | 201 `{ message: '注文完了' }` |
| エラー | 400 `{ error: '注文情報が不正です' }` |

### 4-5. 注文履歴取得 GET /api/orders

| 項目 | 内容 |
|------|------|
| クエリパラメータ | userId |
| 成功レスポンス | 200 `{ orders: [...] }` |
| エラー | 400 `{ error: 'ユーザーIDが必要です' }` |

### 4-6. 注文キャンセル DELETE /api/orders/[orderId]

| 項目 | 内容 |
|------|------|
| パスパラメータ | orderId |
| 成功レスポンス | 200 `{ message: 'キャンセル完了' }` |
| エラー | 400 `{ error: '注文IDが必要です' }` |

## 5. 意図的に仕込んだバグ一覧

### バグ①：上書きバグ（注文履歴画面）

| 項目 | 内容 |
|------|------|
| 対象画面 | 注文履歴画面 |
| 再現手順 | ①商品Aを注文する → 注文履歴に表示される ②商品Bを注文する → 商品Aの履歴が消え、商品Bのみになる |
| 原因 | POST /api/orders にて、INSERT前に既存注文をDELETEしているため |
| 本来の動作 | 両方の注文が履歴に残る |
| 本来の正しい実装 | INSERT前のDELETE処理を削除する |

### バグ②：境界値バグ（注文確認画面）

| 項目 | 内容 |
|------|------|
| 対象画面 | 注文確認画面 |
| 再現手順 | ①合計金額がちょうど3000円になるように商品を選択する ②注文確認画面で送料300円が表示される |
| 原因 | 送料判定が `subtotal > 3000` になっており、3000円ちょうどが無料判定に入らないため |
| 本来の動作 | 3000円以上で送料無料になる |
| 本来の正しい実装 | `subtotal > 3000` を `subtotal >= 3000` に修正する |

### バグ③：flaky test（テストコード）

| 項目 | 内容 |
|------|------|
| 対象 | Playwrightテストコード |
| 内容 | 意図的に不安定なテストを作成し、安定化させる過程を示す |
| 目的 | テスト自動化の品質・安定性への意識をアピールする |
| 実装状況 | テスト実装フェーズで対応予定 |
